## Integrated Documentation System

Generate an mdBook site from narrative docs/ and an OpenAPI reference.

### Pipeline

```
docs/*.md                  (narrative, hand-written)
docs/mdbook/               mdBook source (book.toml, SUMMARY.md, index.md)
                           + openapi-reference.md (tracked)
                           + copies of docs/ (gitignored, regenerated)
backend/openapi.json       widdershins → docs/mdbook/src/openapi-reference.md
```

Built by `just docs-build` into `target/docs/book/`.

### Directory layout

```
docs/mdbook/              ← mdBook source (tracked)
  book.toml
  src/
    SUMMARY.md              ← tracked
    index.md                ← tracked
    openapi-reference.md    ← tracked (generated, kept for history)
    *.md                    ← gitignored, copied from docs/ during build

docs/                     ← hand-written narrative markdown (tracked)

target/docs/book/         ← build output (gitignored)
```

### Decisions

- mdBook with default theme
- Only the OpenAPI reference is generated (via widdershins); it is committed to git
- `docs/*.md` copies into `docs/mdbook/src/` are gitignored (source of truth is docs/)
- cargo doc and typedoc removed — generated Rust/TypeScript API docs were not useful
- openapi-reference.md is tracked for history; regenerated by `just docs-openapi`
